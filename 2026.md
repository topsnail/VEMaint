# 任务：为车辆管理系统进行全面 UX/UI 重构（直接输出可落地代码）

## 1. 项目背景
- 应用类型：车辆管理系统（工具类 Web 应用）
- 现有技术栈：
  - 前端：React 18 + TypeScript + Vite + Ant Design 5 + Tailwind CSS + React Router
  - 后端/云服务：Cloudflare Pages + Functions + D1 + KV + R2
  - 其他依赖：Hono, JOSE, GSAP, @ant-design/icons
- 当前问题：样式混乱，文字大小不合理、卡片大小不统一、元素间距不一致。需要全面重构 UX/UI。
- 重构目标：
  - 全面中文化（界面、提示、日期格式等）
  - 页面简洁，风格统一，只使用亮色主题，**完全禁止暗色主题**
  - 适合 2K 笔记本电脑显示器（2560×1440，兼顾 125%~150% 缩放）
  - **偏紧凑风格**（不是宽松，信息密度适中，适合工具类应用）
  - 同时适配桌面端（大屏）和手机端（响应式）

## 2. 设计规范（必须严格遵守）

### 2.1 间距系统（基于 8px 网格）
主要间距值（单位 px）：4, 8, 12, 16, 20, 24, 32, 40, 48  
应用规则：
- 卡片内边距：默认 16px（紧凑），特殊卡片可用 24px
- 相邻区块间距：16px ~ 20px
- 页面左右安全边距：桌面 24px，手机 16px
- 表格单元格内边距：上下 12px，左右 16px → 行高 48px
- 表单垂直间距：20px

### 2.2 字体系统
| 用途 | 字号 | 行高 | 字重 |
|------|------|------|------|
| 页面标题 | 24px | 1.4 | 600 |
| 卡片标题 | 18px | 1.5 | 500 |
| 正文/表格内容 | 16px | 1.5 | 400 |
| 辅助文字/标签 | 14px | 1.5 | 400 |
| 按钮文字 | 16px | 1.2 | 500 |

### 2.3 组件尺寸（Ant Design 定制）
- 按钮高度：桌面 40px，手机 36px
- 输入框/选择器高度：桌面 40px，手机 36px
- 表格行高：48px（最小）
- 卡片圆角：8px
- 按钮圆角：8px

### 2.4 颜色（仅亮色）
- 页面背景：`#F5F7FA`
- 卡片背景：`#FFFFFF`
- 主要文字：`#1F2937`
- 次要文字：`#6B7280`
- 边框/分割线：`#E5E7EB`
- 主色：保持 Ant Design 默认 `#1677FF`
- **禁用任何暗色主题变量**：不要 `dark:` 类，不要暗色算法，不要 `prefers-color-scheme` 媒体查询

### 2.5 响应式断点
- 桌面（≥1280px）：内容区最大宽度 1400px，水平居中，左右 padding 24px
- 平板（768px~1279px）：最大宽度 100%，左右 padding 20px
- 手机（<768px）：左右 padding 16px，卡片内边距 16px，按钮高度 36px，表单垂直布局

## 3. 中文化要求
- 使用 `react-i18next` 配置默认语言 `zh-CN`，不需要语言切换器。
- Ant Design 组件通过 `ConfigProvider` 设置 `locale={zhCN}`。
- 日期格式：`YYYY年MM月DD日`，时间格式：`HH:mm`。
- 数字格式化：使用中文逗号分隔或直接显示。
- 所有用户可见文案（按钮、标签、提示、表格列名、表单 placeholder、错误信息）必须为中文。



## 5. 代码约束
- **不要修改任何业务逻辑**（数据获取、状态管理、表单提交、路由跳转等保持原样）。
- 优先使用 Ant Design 组件原生 props（如 `size="middle"`、`type="primary"`），Tailwind 仅用于布局和间距。
- 避免使用内联 `style`。
- 确保移动端触摸区域至少 36px（紧凑可接受 36px，但推荐 40px）。
- 不要引入新的第三方 UI 库。

## 6. 额外要求
- 所有输出代码必须**完整、可直接复制粘贴到现有项目中**，不要省略号或注释占位。
- 提供**手机端适配说明**（如何在 MainLayout 中实现响应式侧边栏/底部导航）。
- 提供**验证清单**（检查重构是否完成，包括 2K 屏幕、手机模拟器、暗色主题残留等）。

## 7. 重要提醒
- 当前项目已使用 React Router，MainLayout 需包含 `<Outlet />`。
- 如果项目中没有 `react-i18next`，请在输出中说明需要安装的依赖：`npm install i18next react-i18next`。
- 不要生成暗色主题相关代码。
- 保持偏紧凑风格，避免过大留白。

请直接帮我优化落地
